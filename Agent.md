# Agent.md

## 项目定位

- 本仓库用于构建 `aichina.me` 的新网站。
- 这不是个人主页，不是简历站，也不是学术模板站。
- 这是一个去个人化的主题网站，核心气质是：中国语境、人工智能、现代东方审美、克制表达。
- 站点应更像一个命题空间、展览首页或未来感刊物首页，而不是“关于我”的展示页面。

## 核心方向

- 去个人化：不突出个人身份，不围绕“我是谁”组织页面。
- 去简历化：不使用教育经历、工作经历、奖项、论文列表驱动首页结构。
- 去社交化：不放个人联系方式、社交账号、即时沟通入口。
- 去模板化：不使用常见学术主页、博客主题、产品落地页的套板逻辑。
- 强视觉统一：所有页面必须服务于同一套审美语言。

## 明确禁止

- 禁止出现姓名、头像、个人简介、个人履历、教育经历、工作经历、奖项、联系方式。
- 禁止出现邮箱、电话、微信、GitHub、Google Scholar、社交媒体入口。
- 禁止做成“个人主页”“学术主页”“简历页”“作品集主页”。
- 禁止为了凑内容加入占位段落、假栏目、假论文、假项目、假链接。
- 禁止出现廉价中国风元素，例如龙纹、灯笼、祥云、红金婚庆风、政府官网风。
- 禁止大面积滥用纯红、纯黑、纯金，避免压迫感和俗气。
- 禁止使用“兜底策略”掩盖设计问题。做不出来时要指出问题，不要退回普通模板。

## 内容原则

- 首页以主题表达为主，不以人物介绍为主。
- 文案应短、准、克制，接近宣言、标题、短句，而不是自我介绍。
- 页面内容优先使用抽象栏目，例如 `Statement`、`Archive`、`Signals`、`Fragments`、`Language`、`Systems`、`Reality`。
- 若后续增加内容，也应围绕主题站逻辑扩展，而不是回到个人资料页结构。
- 所有内容必须服务于统一命题：`AI in China` 的审美、语境与思考空间。

## 视觉方向

- 总体风格：现代、安静、东方、克制、未来感、刊物感。
- 目标体验：像一本未来感很强的东方印刷物，而不是“中国风网页”。
- 页面应保留足够留白，避免信息堆砌。
- 排版应有仪式感和秩序感，层级清晰，但不做传统门户式布局。

## 固定色板

以下颜色作为本项目当前默认配色，未经明确决策不要擅自更换：

```css
:root {
  --cinnabar: #C02C38;
  --seal-red: #9D1F2B;
  --deep-red: #781722;
  --silk-white: #F7F3EB;
  --paper-shadow: #E8DED0;
  --ink-brown: #211A18;
  --charcoal: #3A312E;
  --bronze: #92714A;
  --text: #201B19;
  --text-muted: #6E625B;
  --line: rgba(157, 31, 43, 0.16);
}
```

## 配色使用规则

- 背景主色优先使用 `--silk-white`。
- 主标题、关键强调、局部结构线使用 `--cinnabar`。
- 交互态和悬停态优先使用 `--seal-red`。
- 深色区块使用 `--ink-brown` 或 `--charcoal`。
- `--bronze` 仅允许极少量点缀，不得成为主色。
- 红色只能作为关键视觉切口，不能大面积铺满页面。

## 设计限制

- 不允许页面出现“模板味”很重的 Hero + 自我介绍 + 社交按钮组合。
- 不允许默认科技风蓝紫渐变。
- 不允许用花哨动效掩盖布局平庸。
- 不允许使用过多图标、卡片、徽标和装饰性边框。
- 不允许页面像企业官网、学校官网、简历页、博客首页。

## 允许的视觉元素

- 细线条。
- 留白。
- 网格。
- 轻微纸张质感。
- 极弱噪点。
- 克制的分割线。
- 缓慢、轻量的动效。
- 有东方印刷感但不过度象征化的图形语言。

## 开发原则

1. 先保证方向正确，再写代码。
2. 每次改版都必须检查是否偏离“去个人化”和“非简历化”目标。
3. 如果某个设计方案让页面重新变得像模板站，应直接否决。
4. 如果某个内容区块无法自圆其说，应删除，不要硬凑。
5. 若需要新增页面或模块，先说明其在整体叙事中的必要性。

## 交付要求

- 说明本次新增或修改服务于什么视觉目标。
- 说明是否符合当前 `Agent.md` 的约束。
- 若存在妥协，必须明确指出妥协点，不得隐瞒。
