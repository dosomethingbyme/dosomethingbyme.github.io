# Agent.md

## 项目定位

- 这是一个基于 Jekyll / GitHub Pages 的个人学术主页仓库，来源于 `acad-homepage.github.io` 的二次开发版本。
- 中文主页内容主要在 `_pages/about.md`。
- 英文主页内容主要在 `_pages/en/about.md`。
- 站点配置在 `_config.yml`。
- 样式主入口在 `assets/css/main.scss`。
- Google Scholar 相关抓取脚本在 `google_scholar_crawler/main.py`。
- 本地启动方式是 `bash run_server.sh`，实际执行 `bundle exec jekyll serve --livereload`。

## 基本工作原则

1. 先读相关文件，再做修改，不允许脱离上下文直接重写页面或样式。
2. 个人信息、教育经历、论文、奖项、单位、链接、DOI、引用数据必须以仓库现有内容或用户明确提供的信息为准。
3. 中文页与英文页涉及同一事实时，必须保持语义一致，不允许一边更新、一边遗漏。
4. 修改应延续当前站点已经形成的个人化学术主页风格，不要回退到模板默认外观。
5. 任何改动完成后，都应优先验证是否会影响 Jekyll 构建或页面渲染。

## 明确禁止：不要使用兜底策略

本项目明确禁止任何“兜底式”处理。不要为了“看起来能用”而掩盖问题、伪造内容或静默降级。

### 禁止事项

- 信息缺失时，擅自补默认文案、占位内容、假论文、假链接、假 DOI、假引用数。
- 外部数据失败时，用写死的静态数字或伪造结果冒充真实数据。
- 资源缺失时，自动替换成无关图片、无关图标、无关模块，只为避免报错或空白。
- 中英文内容不一致时，凭猜测补齐翻译或凭空扩写事实。
- 样式或交互改不动时，直接退回模板默认实现，且不说明原因和影响范围。
- 无法确认需求时，自行决定一个“差不多”的实现交差。

## 正确处理方式

- 缺少信息：保留现状，或显式标注待补充项，并向用户说明缺口。
- 缺少资源：指出具体缺失文件或路径，不用无关资源替代。
- 抓取失败或接口失败：保留失败事实，排查原因，不伪造成功结果。
- 事实不确定：先向用户确认，或者明确说明无法确认的部分。
- 需要临时下线某功能：必须写清楚原因、影响范围和恢复条件。

## 本仓库的重点注意事项

- 论文、教育经历、奖项等内容属于高敏感事实信息，禁止主观润色成新的事实。
- Google Scholar 数据来自仓库外部流程，若数据缺失，应先检查抓取链路，不要在页面里硬编码“看起来正常”的数字。
- 页面内容使用 Markdown + HTML 混写，修改时要注意不要破坏现有结构。
- 样式集中在 `assets/css/main.scss`，优先在现有视觉体系内增量调整，不要大范围推翻。

## 交付要求

- 说明改了什么。
- 说明如何验证。
- 如果存在未解决项，明确列出，不要用兜底方案掩盖。
